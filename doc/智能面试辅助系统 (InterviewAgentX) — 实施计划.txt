 智能面试辅助系统 (InterviewAgentX) — 实施计划

 Context

 基于 prompt.txt 的完整规格，从零构建一个智能面试辅助 AI 应用。技术栈：Python + LangChain + LangGraph + Milvus +
 PostgreSQL + DeepSeek API + RAG + Multi-agent，后端 FastAPI，前端 Vue 3 + TypeScript，Docker Compose 部署。

 核心架构设计

 1. LangGraph 状态管理 (不使用协调Agent)

 图拓扑本身就是协调者：
 - 线性流程用 add_edge 硬连接（简历上传→OCR→简历分析）
 - 动态决策用 conditional_edges（是否还有下一题？→继续作答或生成报告）
 - 人工介入(HITL)：process_current_answer 节点等待前端POST答案后恢复执行

 # 图拓扑
 START →resume_analysis →generate_questions →process_current_answer
                                                     ↑       ↓
                                           [more questions]  analyze_current_answer
                                                                     ↓
                                                          [all done] →generate_report →END

 状态使用 TypedDict + Annotated[list, operator.add] 归约器实现增量追加，每个Agent输出结构化JSON而非长文本。

 2. 两种作答评估模式

 ┌───────────────┬────────────────┬───────────────────────────────────────────────────────────────────┐
 │     题型      │    评估模式    │                             实现方式                              │
 ├───────────────┼────────────────┼───────────────────────────────────────────────────────────────────┤
 │ 知识库题目    │ RAG混合检索    │ Milvus召回标准答案 →向量相似度 →LLM判定得分点覆盖               │
 ├───────────────┼────────────────┼───────────────────────────────────────────────────────────────────┤
 │ 简历/项目题目 │ LLM-as-a-Judge │ Prompt含{简历片段+题目+回答} →从真实性/逻辑性/技术深度三维度打分 │
 └───────────────┴────────────────┴───────────────────────────────────────────────────────────────────┘

 3. 数据一致性 (PostgreSQL ↔Milvus)

 - Milvus Entity ID 直接使用 PostgreSQL kb_items.id (UUID →VARCHAR(36))
 - 所有表使用 is_deleted 软删除
 - 知识库双写：先更新Milvus，成功后提交PG事务
 - Milvus检索始终带 expr="is_deleted == false" 过滤

 4. WebSocket 流式 STT

 - 前端 Web Audio API →WebSocket 实时发送音频切片 →后端对接流式STT引擎
 - 服务端推送 {type: "partial"|"final"|"error"|"end", text, ...} JSON帧
 - 求职者说话同时，文本同步显示在面试官屏幕上

 数据库设计 (PostgreSQL)

 核心表：candidates, resumes, resume_analyses, kb_categories, kb_items, interview_sessions, questions, answers,
 answer_analyses, interview_reports

 - 所有表UUID主键 + is_deleted 软删除 + created_at/updated_at
 - kb_items 表含 embedding_model, embedding_dim, is_vectorized, version 字段
 - interview_sessions 含 langgraph_thread_id 关联LangGraph checkpointer
 - answer_analyses 含 eval_mode 字段区分评估模式，analysis_json 存储结构化结果

 Milvus Collection

 - Collection: knowledge_base_vectors，主键 id (VARCHAR 36) = PG kb_items.id
 - 稠密向量 answer_vector (1536维, COSINE相似度)
 - 稀疏向量 question_sparse_vector (BM25)
 - 标量过滤字段：category_id, tags, difficulty, version, is_deleted

 API 端点

 REST (前缀 /api/v1)

 - POST/GET /resumes/upload|{id}|{id}/analysis —简历模块
 - POST/GET /interviews|{session_id}/start|questions|answers —面试模块
 - GET /reports/{session_id} —报告模块
 - CRUD /knowledge-base/items|categories —知识库模块
 - GET /records|{candidate_id} —面试记录模块
 - CRUD /candidates —候选人模块

 WebSocket

 - /ws/stt/{session_id}/{question_id} —二进制音频入 / JSON文本出

 Docker Compose

 7个服务：postgres(pgvector/pg17), etcd, minio, milvus(standalone 2.5), backend(FastAPI), frontend(Nginx+Vue),
 milvus-init

 - Milvus 限制 6GB 内存 / 4 CPU
 - Backend 限制 3GB 内存 / 3 CPU
 - 通过 HTTP_PROXY/HTTPS_PROXY 环境变量支持中国大陆网络代理

 实施阶段

 Phase 0: 项目脚手架

 - 创建项目结构、Poetry配置、Vue+Vite初始化、Docker Compose、Alembic迁移、Milvus初始化脚本

 Phase 1: 数据与服务层

 - SQLAlchemy ORM模型、Pydantic Schema、MinIO/OCR/Embedding/Milvus/KB Service

 Phase 2: Agent与LangGraph

 - InterviewState定义、AgentFactory(DeepSeek)、5个Prompt模板、4个Graph Node + Routing
 Node、RAG混合检索器、LangGraph编译+Checkpointer

 Phase 3: API与WebSocket

 - FastAPI路由、依赖注入、STT Service、WebSocket端点、中间件配置

 Phase 4: 前端

 - Axios客户端、TypeScript类型、Pinia stores、Composables(useWebSocket/useAudioRecorder)、所有页面和组件

 Phase 5: 集成与部署

 - Nginx配置、Dockerfile多阶段构建、E2E测试、代理配置验证、资源调优

 验证方案

 1. docker compose up -d 启动全部服务，验证健康检查通过
 2. 上传简历PDF →验证OCR解析 →查看简历分析结果
 3. 知识库CRUD →验证Milvus向量同步(插入/更新/删除)
 4. 创建面试会话 →生成题目 →WebSocket录音 →转写文本 →作答分析
 5. 结束面试 →生成面试报告
 6. 面试记录列表查询
